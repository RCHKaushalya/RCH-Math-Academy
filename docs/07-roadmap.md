# Project Roadmap

Document Status: Draft  
Prepared On: 2026-06-28  
Project: Math Tutor Web Application

## 1. Roadmap Summary

The project should be built in phases. The first goal is not monetization. The first goal is to create a useful, original, searchable math learning platform. Monetization should follow after the site is ready for public review.

The long-term curriculum target is from nursery-level early numeracy through research-level mathematics. The implementation must still begin with a focused, high-quality foundation.

## 2. Phase 0: Planning and Repository Setup

Target Duration: 1 to 3 days

Deliverables:

- Formal documentation in `docs`.
- Master action tracker.
- Curriculum scope document.
- Design and content standards document.
- Git repository initialized.
- GitHub repository created.
- Project README created.
- Initial issue list created.
- Technology stack confirmed.
- Level-specific pedagogy strategy confirmed.

Exit Criteria:

- The project has a documented plan.
- The repository is ready for implementation.

## 3. Phase 1: MVP Application Foundation

Target Duration: 3 to 7 days

Deliverables:

- Dependency-free static MVP.
- Future-ready plan for Next.js, TypeScript, and Tailwind CSS after npm install is reliable.
- Base layout.
- Responsive navigation.
- Home page.
- Topic index page.
- Lesson page template.
- Early learning activity template.
- Policy page templates.
- Gray elegant light theme.

Exit Criteria:

- The site opens locally through `index.html`.
- The static check passes.
- Basic navigation works on mobile and desktop.

## 4. Phase 2: First Content Release

Target Duration: 1 to 3 weeks

Deliverables:

- At least 30 original lesson pages.
- At least 5 topic groups.
- Early numeracy, arithmetic, algebra, geometry, and statistics seed coverage.
- At least 2 nursery activities that do not require reading.
- Worked examples for each lesson.
- Practice questions for each lesson.
- Internal links between lessons.
- Sitemap and robots file.

Exit Criteria:

- The site has enough useful content to be valuable without ads.
- Each page has unique SEO metadata.

## 5. Phase 3: Interactive Practice

Target Duration: 1 to 2 weeks

Deliverables:

- Multiple choice question component.
- Numeric answer component.
- Answer feedback.
- Step-by-step solution reveal.
- Practice progress within a page.

Exit Criteria:

- Learners can practice and receive useful feedback without creating an account.

## 6. Phase 4: Public Launch

Target Duration: 2 to 5 days

Deliverables:

- Production hosting configured.
- Custom domain connected.
- HTTPS enabled.
- Google Search Console configured.
- Analytics configured.
- Accessibility pass.
- Performance pass.

Exit Criteria:

- The site is publicly available and indexable.

## 7. Phase 5: AdSense Readiness

Target Duration: 1 to 2 weeks after public launch

Deliverables:

- About page.
- Contact page.
- Privacy Policy.
- Terms of Use.
- Educational disclaimer.
- Content quality review.
- Ad placement plan.
- AdSense application.

Exit Criteria:

- The site is submitted for AdSense review only after it has meaningful original content and required policy pages.

## 8. Phase 6: Growth and Improvement

Target Duration: Ongoing

Deliverables:

- More topics and levels.
- Better practice tools.
- Search improvements.
- Worksheet downloads.
- Performance monitoring.
- Content updates based on search data.

Exit Criteria:

- The platform grows through consistent publishing and measured improvements.

1. Initialize the Git repository.
2. Create the GitHub repository.
3. Initialize the application stack.
4. Implement the gray elegant light theme.
5. Create level-specific page templates.
6. Create the first nursery activities.
7. Create the first topic and lesson templates.
8. Publish the first 5 to 10 lessons.
9. Set up free deployment.
