# Product Requirements Document

Document Status: Draft  
Prepared On: 2026-06-28  
Project: Math Tutor Web Application

## 1. Product Vision

The platform will help learners at different mathematics levels study for free through structured lessons, worked examples, interactive practice, and clear learning paths. The long-term scope is broad: it should grow from nursery-level early numeracy to advanced mathematics useful for university students, PhD researchers, engineers, scientists, and NASA-level research users.

The first product priority is math tutoring quality. Monetization through AdSense is a later phase after the site is useful, original, public, and policy-ready.

The product must not use one teaching voice for every learner. Each level needs a different teaching style, interaction model, and content depth. Nursery learners should not be expected to read; their section must be visual, audio-guided, touch-friendly, and caregiver-friendly.

## 2. Target Users

| User Group | Need |
| --- | --- |
| Primary students | Simple explanations, visuals, and practice. |
| Middle school learners | Foundations in arithmetic, fractions, geometry, and algebra. |
| High school learners | Exam-focused lessons and step-by-step examples. |
| College beginners | Refresher lessons and advanced foundations. |
| Adult learners | Practical math, confidence building, and self-paced review. |
| Parents and tutors | Printable practice and reliable explanations. |
| Undergraduate learners | Calculus, linear algebra, statistics, discrete math, and modeling support. |
| Graduate learners | Proof-based topics, advanced probability, analysis, algebra, and applied math. |
| Research-oriented users | Numerical methods, optimization, control, simulation, and aerospace-relevant mathematics. |
| Nursery learners | Audio-first, picture-based, touch-friendly activities that do not require independent reading. |

## 3. Product Goals

- Provide free math learning materials for multiple levels.
- Make lessons easy to discover from search engines.
- Make practice interactive and immediate.
- Build enough original content for AdSense eligibility.
- Keep the platform fast, accessible, and mobile-friendly.
- Use free tools and hosting where practical.
- Establish a curriculum path that can grow from early learning to research-level mathematics.
- Use a modern, elegant, sky-blue light theme.
- Maintain a high content quality standard before monetization.
- Provide different lesson templates for pre-reading, school, university, and research learners.

## 4. MVP Scope

The MVP should include:

- Home page with level-based navigation.
- Topic pages for core math areas.
- Lesson pages with explanations and examples.
- Practice questions with instant feedback.
- Search or topic index.
- About, Contact, Privacy Policy, Terms, and Disclaimer pages.
- SEO metadata for each lesson.
- Responsive design for mobile and desktop.

## 5. Learning Levels

Initial level structure:

- L00: Nursery and early numeracy.
- L01: Primary foundations.
- L02: Upper primary.
- L03: Middle school.
- L04: High school foundations.
- L05: Advanced high school and exam preparation.
- L06: College readiness.
- L07: Undergraduate core mathematics.
- L08: Advanced undergraduate mathematics.
- L09: Graduate mathematics.
- L10: Research and specialist mathematics.

The platform should allow new levels to be added without changing the full application architecture.

Nursery and early numeracy should use a dedicated early learning activity mode rather than a standard article-style lesson page.

## 6. Core Features

### 6.1 Lessons

Each lesson should include:

- Learning objective.
- Short explanation.
- Worked examples.
- Common mistakes.
- Practice questions.
- Summary.
- Related lessons.

Lesson structure must vary by level. Advanced lessons should support formal definitions and proofs. Nursery activities should use visual tasks, audio prompts, and one action per screen.

### 6.2 Practice

Practice should include:

- Multiple choice questions.
- Numeric answers.
- Step-by-step worked solutions.
- Feedback after each answer.
- Difficulty labels.

### 6.3 Search and Navigation

The site should support:

- Topic browsing.
- Level browsing.
- Related lesson links.
- Search engine friendly URLs.
- Breadcrumb navigation.

### 6.4 Accessibility

The site should support:

- Keyboard navigation.
- Clear color contrast.
- Readable font sizes.
- Screen-reader friendly headings.
- Avoiding content hidden only in images.

## 7. Non-Functional Requirements

| Requirement | Target |
| --- | --- |
| Performance | Pages should load quickly on mobile networks. |
| SEO | Each lesson should have title, description, headings, and structured content. |
| Maintainability | Lessons should be stored as version-controlled content files. |
| Security | No sensitive user data in the MVP. |
| Reliability | Static pages should continue working without a backend. |
| Cost | Initial hosting and tooling should remain free where possible. |

## 8. Out of Scope for MVP

The following should not be included in the first release:

- Paid subscriptions.
- User accounts.
- Live chat.
- AI tutor requiring paid API usage.
- Large video hosting.
- Complex admin dashboard.
- Full adaptive learning engine.

These features can be considered after the content platform proves useful.

## 9. Success Criteria

MVP success will be measured by:

- At least 30 original lesson pages published.
- At least 5 topic groups available.
- Mobile Lighthouse performance above 90 where possible.
- All required policy pages completed.
- Site indexed by Google Search Console.
- AdSense application submitted only after the site is content-ready.
